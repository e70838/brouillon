Affichage du résultat d'un calcul (erroné dans le livre p102, mais corrigé par internet)

Les attributs name dans les forms sont pratiques pour accéder aux objets. Par exemple si la form a pour nom "f1", on peut coder : 


        document.addEventListener("input",
          function(e) {
            if (document.forms.f1 == e.target.form)
              e.target.form.calcul.value =
                parseFloat(e.target.form.num1.value) * parseFloat(e.target.form.num2.value);
        }, true);

C'est marrant comment les champs gardent leur valeur sur un refresh

feuilles de styles

Pour les feuilles externes, il faut écrire


        <link rel="stylesheet" href="style.css">

Le sélecteur p.chapeau s'applique aux éléments p ayant la classe chapeau. Pour les id, on utilise #.

Il existe des pseudo-classes :hover :link(lien non visité) :visited(lien visité) :active(?) :first-child :focus(qui a le focus) :lang(fr)

Il existe des pseudo-éléments avec ::, par exemple p::first-letter {font-size: 150%;}
::before ::after

On peut combiner avec la virgule (,) pour faire des "ou". Avec un espace pour indiquer une hierarchie (div p signifie un paragraphe dans un div).
Avec un > pour indiquer une imbrication directe.
Avec un + pour indiquer une juxtaposition h1 + h2 {} s'applique à chaque h2 qui suit un h1.

Sélecteur d'attribut : input[type=button] {} s'applique aux input de type button

Pour la priorité, on regarde d'abord si c'est local ou important, ensuite on regarde si il y a un id, ensuite, on compte le nombre de classes, ensuite le nombre d'éléments.

  • vmaxplus grande valeur de vw et vh
